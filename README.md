# FilmFlip - Рандомайзер фильмов 🎬

**Не знаешь что посмотреть? Доверься случаю!**

FilmFlip - это веб-приложение для случайного выбора фильмов с использованием TMDb API. Сайт полностью адаптирован для мобильных устройств и готов к размещению на GitHub Pages.

## ✨ Возможности

- 🎲 **Случайный выбор фильма** с учётом ваших предпочтений
- 🎭 **Фильтрация по жанрам** (все жанры из TMDb)
- 📅 **Выбор десятилетия** (1950-е - 2020-е)
- ⭐ **Фильтр по рейтингу** (0-10)
- 🎬 **Просмотр трейлера** на YouTube
- 📺 **Где смотреть** - информация о стриминговых платформах в России
- 📤 **Поделиться** - скопируйте рекомендацию для друзей
- 🌙 **Тёмная тема** - кинематографичный дизайн
- 📱 **Mobile-first** - отлично работает на телефонах
- 🔍 **Умный поиск** - 3 попытки с расширением фильтров + fallback

## 🚀 Быстрый старт

### 1. Получите API ключ TMDb (бесплатно)

1. Перейдите на [https://www.themoviedb.org/](https://www.themoviedb.org/)
2. Зарегистрируйтесь (если нет аккаунта)
3. Перейдите в настройки профиля → **API**
4. Нажмите "Create" или "Request an API Key"
5. Выберите "Developer" (для личного использования)
6. Заполните форму (можно кратко описать проект)
7. После одобрения скопируйте **API Key (v3 auth)**

> ⏱ Обычно ключ выдаётся мгновенно или в течение нескольких минут.

### 2. Настройте проект

#### Вариант A: Вставка API ключа в код (рекомендуется для GitHub Pages)

1. Откройте файл `script.js`
2. Найдите строку в начале файла:
   ```javascript
   const API_KEY = 'YOUR_TMDB_API_KEY_HERE';
   ```
3. Замените `'YOUR_TMDB_API_KEY_HERE'` на ваш API ключ:
   ```javascript
   const API_KEY = 'ваш_ключ_здесь';
   ```
4. Сохраните файл

#### Вариант B: Через переменную окружения (для локальной разработки)

1. Скопируйте `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```
2. Отредактируйте `.env` и вставьте ваш API ключ

### 3. Размещение на GitHub Pages

#### Вариант A: Через GitHub UI

1. Создайте новый репозиторий на GitHub
2. Загрузите файлы проекта (`index.html`, `style.css`, `script.js`, `README.md`, `.env.example`, `.gitignore`)
3. Перейдите в **Settings** → **Pages**
4. В разделе "Source" выберите ветку `main` и папку `/ (root)`
5. Нажмите **Save**
6. Через 1-2 минуты сайт будет доступен по адресу: `https://ваш-username.github.io/имя-репозитория/`

#### Вариант B: Через Git командную строку

```bash
# Инициализация репозитория (если ещё не инициализирован)
git init
git add .
git commit -m "Initial commit"

# Добавьте удалённый репозиторий
git remote add origin https://github.com/ВАШ_USERNAME/ВАШ РЕПОЗИТОРИЙ.git

# Отправьте код
git push -u origin main
```

Затем включите GitHub Pages в настройках репозитория.

### 4. Использование

1. Откройте сайт в браузере
2. Настройте фильтры (жанр, десятилетие, рейтинг)
3. Нажмите **"🎲 Найти фильм"** или **"⚡ Мне повезёт!"**
4. Смотрите информацию о фильме, трейлер и где посмотреть!

## 🔍 Как работает умный поиск

При выборе узких фильтров (например, "Ужасы 2020-х с рейтингом 8+") система автоматически делает 3 попытки:

1. **Попытка 1**: С точными фильтрами (жанр + год + рейтинг)
2. **Попытка 2**: Без ограничения по рейтингу (жанр + год)
3. **Попытка 3**: Только по жанру (игнорируя год и рейтинг)

Если после 3 попыток ничего не найдено — показывается дружелюбное сообщение с советами или случайный популярный фильм.

## 📝 Примечания

- API ключ можно сохранить прямо в `script.js` для удобства развёртывания на GitHub Pages
- Для корректной работы необходим доступ к TMDb API
- Информация о стриминговых платформах доступна для региона RU
- Некоторые фильмы могут не иметь трейлеров или информации о просмотре
- Минимальное количество голосов: 50 (чтобы не показывать неизвестные фильмы)
